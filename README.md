# SoFake — Fake News Evolution Simulator

SoFake is an agent-based simulation framework built for UNSW COMP3900. It models how a factual news story distorts as it propagates through a social network of LLM-driven agents, then quantifies the deviation between the ground truth and every evolved post using the FUSE framework.

Given a ground-truth article, the system:
1. Spawns a configurable population of agents with HEXACO personality profiles and role mixes (spreader / commentator / verifier / bystander).
2. Simulates step-by-step interactions on a clustered + scale-free social graph. Agents like, retweet, quote, comment, and write new posts — text is generated by an LLM conditioned on each agent's personality.
3. Scores every evolved post against the ground truth and its parent post on 9 FUSE dimensions (sentiment shift, new info, certainty shift, stylistic shift, temporal shift, perspective deviation, sensationalism, source attribution, political bias).
4. Renders drift graphs, per-step metrics, and per-agent FUSE reports in a React dashboard.

## Architecture

Four Docker services orchestrated by `docker-compose`:

| Service    | Tech                        | Host port | Purpose                                                 |
|------------|-----------------------------|-----------|---------------------------------------------------------|
| `frontend` | React 19 + Vite + nginx     | `80`      | SPA dashboard                                           |
| `backend`  | FastAPI                     | `9000`    | Orchestrates simulation and FUSE scoring                |
| `agent`    | FastAPI + Groq (Llama 3.1)  | `9001`    | Social-graph + agent behaviour simulation               |
| `fuse`     | FastAPI + LLM               | `9002`    | 9-dimension deviation scorer                            |

The frontend proxies `/api/*` to `backend` via nginx. `backend` calls `agent` for the simulation run and `fuse` for per-post scoring.

## Prerequisites

- [Docker Desktop](https://www.docker.com/products/docker-desktop) (includes Docker Compose v2)
- An `ANTHROPIC_API_KEY` — used by the FUSE scoring service
- A `GROQ_API_KEY` — used by the agent simulation service

## Quick Start (customer / marker)

```bash
# 1. Clone the repo
git clone <repo-url>
cd fake_news_project

# 2. Create a .env file in the project root with your API keys.
#    IMPORTANT: no space after "=", LF line endings (not CRLF).
cat > .env <<'EOF'
ANTHROPIC_API_KEY=your_fuse_api_key_here
GROQ_API_KEY=your_agent_api_key_here
EOF

# 3. Build images and start all services (first run ≈ 3–5 min)
docker-compose up --build -d

# 4. Open the dashboard
#    http://localhost
```

### Verify everything is healthy

```bash
curl http://localhost:9000/healthcheck        # backend
curl http://localhost:9002/healthcheck        # fuse    -> {"status":"healthy","scorer_ready":true}
docker-compose ps                             # all services should be "Up"
```

## Using the Dashboard

1. **New Simulation** — paste a factual news article into the Ground Truth box, tweak agent count / steps / seed, and click **Start Simulation**. A run with 30 agents × 60 steps takes roughly 1–3 minutes.
2. **Graph View** — interactive social network graph showing agents and their connections, plus line charts of how emotional charge, controversy, fringe score, and threat level drift across generations.
3. **Overview Dashboard** — posts per step, action breakdown, FUSE deviation bars for each sampled post.
4. **FUSE Comparison** — per-post breakdown: scores vs. the original ground truth and vs. the immediate parent post.
5. **FUSE Report** — per-agent view: each agent's HEXACO personality profile (bar chart), average FUSE deviation scores, and post-by-post breakdown.
6. **Saved Runs** — every completed run is persisted to the database; full post timeline per run.
7. **Export Report** — click the Export Report button (top right) to download the full run log as a JSON file.

## Stopping / Restarting

```bash
docker-compose stop              # stop containers, keep data
docker-compose start             # start them again
docker-compose down              # stop + remove containers (data in ./backend/news.db survives)
docker-compose down -v           # stop + remove + wipe volumes
```

## Troubleshooting

**FUSE pages show empty scores / `fuse_evaluations` is `[]`:**
Usually a FUSE API-key problem. Check FUSE logs — `401 invalid x-api-key` is the telltale sign.
```bash
docker-compose logs --tail=50 fuse
```
Common causes: space after `=` in `.env`, CRLF line endings, revoked/expired key. Fix `.env`, then:
```bash
docker-compose up -d --force-recreate fuse backend
```

**`docker-compose up --build` fails pulling `node:20-alpine` / `python:3.12-slim` with a TLS certificate error:**
Your network (ISP or corporate proxy) is intercepting Docker Hub TLS. Either configure a registry mirror in Docker Desktop → Docker Engine, or switch networks.

**Simulation times out at 10 minutes:**
Reduce `steps` or `agent_count` on the New Simulation page. Default `backend` timeout to `agent` is 600s.

## Notes for Developers

If you're developing locally, see the individual service READMEs (`FUSE/README.md`, `frontend/README.md`). A `docker-compose.override.yml` is typically used to enable source mounts + hot reload — it is gitignored, so customers see only the production compose file.
